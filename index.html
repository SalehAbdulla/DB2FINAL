Unit 10: PL/SQL Functions

Key Concepts

A function is a PL/SQL module that returns a value. Functions are reusable and compiled once, stored in your schema.

Function Structure

FUNCTION [schema.]name [(parameters)]
RETURN return_datatype
IS|AS
  [declarations]
BEGIN
  -- PL/SQL statements
  RETURN value;
[EXCEPTION
  exception_handlers]
END [name];

Important Points

  • Must have a RETURN statement in the executable section
  • Parameters are IN mode only (read-only)
  • Can be called from SELECT statements
  • Formal parameters = placeholders in header
  • Actual parameters = values passed during call
  • Drop with: DROP FUNCTION function_name;

Example: Calculate Age Function

CREATE OR REPLACE FUNCTION Calculate_Age (Birth_Date IN Date) 
RETURN REAL
IS
  Age REAL;
BEGIN
  Age := (SYSDATE - Birth_Date) / 365.25;
  RETURN Age;
EXCEPTION
  WHEN OTHERS THEN 
    RETURN NULL;
END Calculate_Age;

Unit 11: PL/SQL Procedures

Key Concepts

A procedure is a module that performs actions. It doesn't return a value through RETURN (uses OUT/IN OUT parameters instead).

Procedure Structure

PROCEDURE [schema.]name [(parameters)]
IS|AS
  [declarations]
BEGIN
  -- PL/SQL statements
[EXCEPTION
  exception_handlers]
END [name];

Parameter Modes

  • IN: Passes value TO procedure (default) - read-only
  • OUT: Returns value FROM procedure - write-only, no default value
  • IN OUT: Both input and output - can read and write

Calling Procedures

-- Method 1: CALL statement
CALL procedure_name(params);

-- Method 2: EXEC (SQL*Plus)
EXEC procedure_name(params);

-- Method 3: Within PL/SQL block
BEGIN
  procedure_name(params);
END;

Example: Divide Numbers

CREATE OR REPLACE PROCEDURE Divide_Numbers
  (first_number IN INTEGER, 
   second_number IN INTEGER, 
   result OUT NUMBER)
IS
BEGIN
  result := first_number / second_number;
END;

Unit 12: Exception Handling

Key Concepts

An exception is a PL/SQL error raised during execution. Exceptions can be handled to prevent program crashes.

Exception Types

  • Predefined: Oracle-defined (NO_DATA_FOUND, TOO_MANY_ROWS, ZERO_DIVIDE, etc.)
  • Non-predefined: Oracle errors without names (use PRAGMA EXCEPTION_INIT)
  • User-defined: Custom exceptions declared in DECLARE section

Exception Handling Syntax

EXCEPTION
  WHEN exception_name THEN
    -- handle exception
  WHEN OTHERS THEN
    -- handle all other exceptions
    error_code := SQLCODE;
    error_msg := SQLERRM;
END;

Key Functions

  • SQLCODE: Returns numeric error code
  • SQLERRM: Returns error message text
  • RAISE_APPLICATION_ERROR(code, message): Custom errors (-20001 to -20999)

User-Defined Exception Example

DECLARE 
  e_invalid_dept EXCEPTION;
  v_deptno NUMBER := 500;
BEGIN
  UPDATE departments SET department_name = 'Test'
  WHERE department_id = v_deptno;
  
  IF SQL%NOTFOUND THEN
    RAISE e_invalid_dept;
  END IF;
  
  COMMIT;
EXCEPTION
  WHEN e_invalid_dept THEN
    DBMS_OUTPUT.PUT_LINE('Invalid department ID');
END;

Unit 13: Packages

Key Concepts

A package groups related PL/SQL types, variables, constants, exceptions, cursors, and subprograms into a single schema object.

Package Components

  • Specification (Spec): Interface - declares public items
  • Body: Implementation - defines spec items + private items

Package Specification

CREATE OR REPLACE PACKAGE package_name IS|AS
  -- Public declarations
  variable_name datatype;
  PROCEDURE proc_name(params);
  FUNCTION func_name(params) RETURN datatype;
END package_name;

Package Body

CREATE OR REPLACE PACKAGE BODY package_name IS|AS
  -- Private declarations (optional)
  
  -- Procedure implementations
  PROCEDURE proc_name(params) IS
  BEGIN
    -- code
  END;
  
  -- Function implementations
  FUNCTION func_name(params) RETURN datatype IS
  BEGIN
    -- code
    RETURN value;
  END;
END package_name;

Advantages

  • Modularity: Group related constructs
  • Information hiding: Public vs private
  • Performance: Loaded once into memory
  • Overloading: Multiple subprograms with same name

Calling Package Elements

-- Call package procedure
package_name.procedure_name(params);

-- Call package function
v_result := package_name.function_name(params);

-- Access package variable
package_name.variable_name := value;
